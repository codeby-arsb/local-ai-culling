# Local AI Culling - Future Development Policy

Treat Version 1.0 as the stable baseline.
Do not perform any further repository restructuring, architectural refactoring, module relocation, package renaming, or large code cleanup unless fixing a critical bug.

## Semantic Versioning
- **v1.0.x**: Bug fixes, stability improvements, performance optimizations, documentation updates.
- **v1.1**: Workflow improvements, UI/UX refinements, better photographer experience, better configuration options.
- **v2.0**: New AI capabilities, new detection models, new analysis modules, major architectural additions (only if justified).

## Real-World Testing & No Feature Creep
- The priority is using the software on real photography assignments. Future improvements must be driven by actual field experience.
- Whenever incorrect KEEP/REJECT decisions, UI friction, workflow bottlenecks, performance issues, or edge cases are discovered, document them and address them in the appropriate future version.
- Avoid adding features simply because they seem interesting. Every future feature should solve a real problem encountered during actual use.

Version 1.0 is considered feature complete. Treat this repository as a maintained software product.
